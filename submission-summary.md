# Submission Summary

## 1. 서비스명

멍케어노트 MCP

## 2. 한 줄 소개

반려견의 식단, 산책, 일상 상태, 위험 음식 섭취, 사진 관찰, 가족 대화 내용을 정리해 보호자의 관리 행동과 동물병원 상담 준비를 돕는 MCP 서버입니다.

## 3. 문제 정의

보호자는 반려견이 평소와 다른 행동을 보일 때 어떤 정보를 기록해야 하는지, 어느 정도로 주의해야 하는지, 동물병원에 무엇을 전달해야 하는지 판단하기 어렵습니다.

특히 식욕 저하, 묽은 변, 설사, 구토, 무기력, 혈변, 위험 음식 섭취 같은 일상 신호는 가족 대화나 카카오톡 메시지 속에 흩어져 남는 경우가 많습니다. 병원 상담 시에는 증상 시작 시점, 먹은 음식, 구토 여부, 변 상태, 활동량 같은 핵심 정보가 빠지기 쉽습니다.

## 4. 해결 방법

멍케어노트 MCP는 보호자의 자연어 입력을 MCP tool로 구조화합니다.

- 평상시에는 식단, 산책, 물 섭취, 휴식 관리 행동을 안내합니다.
- 이상 신호가 있으면 `normal`, `watch`, `vet_consult`, `urgent`로 위험도를 분류합니다.
- 보호자가 정보를 일부만 입력해도 실패하지 않고, 확인된 정보와 부족한 정보 질문을 생성합니다.
- 위험 상황은 `riskBadge`, `riskLabel`, `immediateAction`, `doNow`, `avoidActions`로 구조화해 응답 상단에서 바로 알아볼 수 있게 합니다.
- 병원 상담 전 보호자가 그대로 복사해 보여줄 수 있는 `vetShareCard.copyableText`를 생성합니다.
- 위험 음식 섭취 상황은 섭취량, 시간, 증상, 사진 정보를 병원 상담용으로 기록합니다.
- 변/피부 사진은 PlayMCP 또는 카카오톡 Agent의 호스트 AI가 이해하고, MCP는 전달받은 `visualNotes`, `observedSigns`, `relatedSymptoms`만 기록하고 구조화합니다.
- 캡처 읽기와 OCR은 호스트 AI가 담당하며, MCP는 전달받은 `chatText`만 분석해 병원 상담용 요약으로 정리합니다.
- MCP 서버는 자체적으로 외부 생성형 AI API를 호출하지 않아 별도 AI API key 없이 실행할 수 있습니다.
- 주요 응답의 `toolChainGuide`가 현재 단계와 다음 tool 후보를 구조화하되, 동물병원 검색은 보호자의 명시적 요청이 있을 때만 제안합니다.
- `dogProfile`에 기존 병원 정보가 있으면 해당 병원 연락, `vetCallScript`, `vetShareCard.copyableText` 준비를 병원 검색보다 우선합니다.
- 농사로/공공데이터 또는 로컬 샘플의 집밥원료 영양성분을 현재 식욕, 변, 구토, 피부 상태와 연결해 원료 비교 기준과 주의점을 제공합니다.

## 5. MCP Tool 목록

- `check_food_safety`: 음식 위험도를 우선 확인하고 요청 시 일치하는 집밥 원료 영양성분을 별도로 요약합니다.
- `analyze_daily_status`: 식욕, 변, 구토, 활동량 등 오늘 상태를 기반으로 위험도를 분류하고, 입력이 부족하면 확인 질문과 임시 판단을 제공합니다.
- `create_daily_care_note`: 오늘 상태와 병원 요약을 생성하고 식단 관련 요청에는 `ingredientSelectionGuide`를 조건부로 포함합니다.
- `recommend_daily_care`: 위험도와 주요 증상을 바탕으로 식단 원료 주의점, 간식, 물 섭취, 산책, 휴식을 안내합니다.
- `create_vet_visit_summary`: 동물병원 상담 시 보여줄 수 있는 증상 요약문을 생성합니다.
- `summarize_pet_chat_for_vet`: 캡처 이미지를 직접 OCR하지 않고, 호스트 Agent가 읽어낸 `chatText`를 바탕으로 반려견 상태와 병원 상담용 요약을 생성합니다.
- `find_nearby_animal_hospitals`: 보호자가 근처·야간·응급·지역 기반 검색을 명시적으로 요청했을 때 동물병원 후보를 안내합니다.
- `classify_pet_symptom`: 보호자의 자연어 증상 표현을 표준 증상명과 카테고리로 정리합니다.
- `record_pet_photo_observation`: 사진 원본을 진단하거나 분석하지 않고, 보호자 또는 호스트 Agent가 제공한 관찰 텍스트를 기록해 이상 징후와 위험도를 구조화합니다.
- `record_food_ingestion_event`: 음식 섭취 상황을 기록하고 위험도 우선 안내, 일치 원료 성분, 병원 상담용 요약을 제공합니다.

모든 위험도 기반 응답은 가능한 경우 `riskPresentation`을 포함합니다. 이를 통해 보호자는 “빠른 상담 권장”, “상담 권장”, “관찰 필요”, “큰 이상 신호 적음”을 한눈에 보고, 바로 확인해야 할 정보와 피해야 할 행동을 함께 확인할 수 있습니다.

주요 병원 상담 관련 응답은 `vetShareCard`도 포함합니다. `vetShareCard.copyableText`는 반려견 정보, 위험도, 주요 증상, 증상 시작 시점, 식욕/변/구토/활동량 상태, 음식 섭취 기록, 추가 확인 정보, 수의사에게 물어볼 질문을 한 번에 정리합니다. 보호자는 당황한 상황에서도 수의사에게 필요한 정보를 빠뜨리지 않고 전달할 수 있습니다.

멍케어노트 MCP는 분석 결과를 단순 JSON이나 긴 설명으로만 제공하지 않고, 카카오톡에서 바로 사용할 수 있는 `kakaoActionText`를 함께 제공합니다. 보호자는 `chatFirstReply`로 현재 상황을 빠르게 이해하고, `familyShareText`를 가족방에 공유하며, `vetCallScript`를 동물병원 전화 상담에 사용할 수 있습니다. 또한 `nextInputExample`과 `whyThisRisk`를 통해 다음 기록 방법과 위험도 판단 근거를 쉽게 이해할 수 있습니다.

이 출력은 `analyze_daily_status`, `create_daily_care_note`, `summarize_pet_chat_for_vet`, `record_food_ingestion_event`, `record_pet_photo_observation`에 적용됩니다. 기존 `riskPresentation`, `vetShareCard`, `safetyMessage`를 유지하면서 카카오톡 실사용 흐름을 보강한 것이 차별점입니다.

멍케어노트 MCP는 `dogProfile` 기반 자동 보정을 통해 보호자가 매번 반려견 이름, 나이, 몸무게를 반복 입력하지 않아도 상태 기록을 이어갈 수 있습니다. 또한 `recentRecords`가 제공되면 오늘 상태를 최근 기록과 비교해 반복 신호, 악화 가능성, 호전 가능성을 `trendSummary`로 정리합니다. 이를 통해 단발성 질의응답이 아니라 매일 축적되는 반려견 건강 기록 보조 경험을 제공합니다.

현재 요청에 명시된 값은 프로필보다 우선하며, 평소 사료와 오늘 먹은 음식은 구분합니다. `knownConditions`와 복용 메모는 보호자 제공 기록으로만 다루고 새로운 질환 판단이나 약 복용 지시에 사용하지 않습니다. 추세 비교 역시 진단이 아니라 병원 상담 준비를 위한 기록 비교 보조입니다.

멍케어노트 MCP는 사진을 직접 진단하지 않고, 보호자 또는 호스트 AI가 제공한 관찰 텍스트를 바탕으로 사진 기록 품질을 평가하고 다음 관찰 가이드를 제공합니다. 사용자는 변·피부 사진 기록 후 다음에 어떤 조건으로 다시 기록해야 하는지, 어떤 변화를 비교해야 하는지, 병원 상담 시 어떤 정보를 함께 전달해야 하는지 안내받을 수 있습니다. 이를 통해 단발성 사진 기록이 아니라 반복 관찰과 병원 상담 준비로 이어지는 실사용 흐름을 제공합니다.

`photoFollowUpGuide`는 관찰 텍스트 충분도, 다음 촬영 방법, 후속 증상 관찰, 비교 포인트, 재기록 권장 여부를 구조화합니다. 사진 픽셀, URL, Base64 원문을 분석하거나 외부 생성형 AI API로 전송하지 않습니다.

멍케어노트 MCP는 위험 상황에서 무조건 새로운 동물병원을 검색하도록 유도하지 않습니다. 보호자가 평소 다니는 병원 또는 `dogProfile`에 등록한 병원 연락을 먼저 안내하고, 병원 검색은 사용자가 명시적으로 요청한 경우에만 `find_nearby_animal_hospitals`로 이어집니다. 기본 행동은 `vetCallScript`와 `vetShareCard.copyableText`로 기존 병원 상담을 빠르게 준비하는 방식이며, `vetContactGuide.shouldAutoSearchHospital`은 항상 `false`입니다. 이를 통해 실제 보호자의 행동 순서에 맞는 Tool Chaining UX를 제공합니다.

멍케어노트 MCP는 반려동물 집밥원료 영양성분 공공데이터를 활용해 반려견의 최근 상태와 프로필에 맞는 원료 선택 기준, 주의 원료, 후보 원료, 급여 전환 시 관찰 포인트를 제공합니다. 단순히 위험 신호를 분류하는 데서 끝나지 않고, 식욕·변 상태·구토·피부 상태 등 일상 기록과 영양성분 데이터를 연결해 보호자가 식단을 더 안전하게 관리할 수 있도록 돕습니다. 이 기능은 진단이나 처방, 치료식 추천이 아니라 수의사 상담과 보호자 기록을 보조하는 방식으로 설계되었습니다.

## 6. 사용 데이터

- 음식 위험도 규칙: MVP rule-based 로컬 규칙
- 일상 상태 위험도 규칙: MVP rule-based 로컬 규칙
- 불완전 입력 처리: 보호자 자연어에서 식욕, 변, 구토, 활동량, 위험 음식 단서를 추정하는 로컬 규칙
- 호스트 AI가 제공한 사진 관찰 텍스트 구조화: rule-based 로컬 규칙
- 호스트 AI가 캡처에서 읽어낸 `chatText` 요약: rule-based 로컬 문자열 규칙
- 동물병원 데이터: 공공데이터포털 동물병원 조회 API 사용 가능
- 동물병원 fallback 데이터: `src/data/animalHospitals.sample.json`
- 증상 사전 fallback 데이터: `src/data/symptomDictionary.sample.json`
- 사진 기록 데이터: `src/data/photoRecords.json`
- 위험 음식 섭취 기록 데이터: `src/data/foodIngestionRecords.json`
- 반려동물 집밥원료 영양성분: 설정 시 농사로/공공데이터 API, 실패 또는 미설정 시 `src/data/petFoodIngredients.sample.json`

공공데이터 API는 키 설정 시 실시간 호출을 시도하고, 실패 시 로컬 샘플 데이터로 fallback합니다.

사진과 캡처 입력에는 외부 생성형 AI API를 사용하지 않습니다. 호스트 AI가 원본에서 추출한 관찰 텍스트만 MCP에 전달하므로 별도 AI API key가 필요하지 않습니다.

## 7. 의료 안전 정책

- 모든 tool 응답에 안전 문구를 포함합니다.
- 안내는 진단이나 처방이 아닙니다.
- 약 복용 지시를 제공하지 않습니다.
- 병원 방문이 불필요하다고 단정하지 않습니다.
- 혈변, 반복 구토, 매우 심한 무기력, 식욕이 전혀 없는 상태, 호흡 이상, 위험 음식 섭취 가능성은 빠른 동물병원 상담 권장으로 안내합니다.
- 위험 상황 표시는 진단이나 처방이 아니라 보호자가 수의사 상담 전 정보를 빠르게 정리하도록 돕는 구조화된 안내입니다.
- 병원 공유용 요약 카드는 보호자 입력 기록을 보기 쉽게 정리한 것이며 진단이나 처방이 아닙니다.
- 사진 기능은 실제 이미지 진단이 아니며, 원본을 직접 분석하지 않고 보호자 또는 호스트 AI가 제공한 관찰 텍스트만 기록합니다.
- 가족 대화/캡처 텍스트 요약 기능은 카카오톡 채팅방을 조회하거나 캡처를 직접 OCR하지 않고 호스트 AI가 제공한 `chatText`만 분석합니다.
- `imageBase64` 원문은 저장하거나 외부 API로 전송하지 않습니다.
- 원료 선택 가이드는 완성 사료, 치료식 또는 급여량 처방이 아니며 한 가지 원료만으로 식사를 구성하도록 안내하지 않습니다.

## 8. 데모 시나리오

1. 보호자가 “강아지가 포도 한 알 먹었어. 몸무게는 11kg이야.”라고 입력합니다.
2. `check_food_safety`가 포도를 위험 음식으로 분류하고 빠른 동물병원 상담 권장을 안내합니다.
3. 보호자가 “우리 강아지가 방금 포도 한 알을 먹은 것 같아요. 몸무게는 11kg이에요.”라고 입력합니다.
4. `record_food_ingestion_event`가 음식명, 섭취량, 시간, 몸무게, 현재 증상, 부족한 정보 질문, 병원 상담용 요약을 기록합니다.
5. 위험 음식 또는 혈변, 반복 구토, 심한 무기력 같은 입력에는 `riskPresentation`이 `🚨 빠른 상담 권장`과 즉시 행동을 응답 상단에 제공합니다.
6. 보호자가 “우리 강아지 6살 11kg인데 어제부터 밥을 반만 먹고 변이 묽어. 구토는 없어.”라고 입력합니다.
7. `create_daily_care_note`가 위험도, 오늘 관리 행동, 병원 상담용 요약을 함께 제공합니다.
8. 보호자가 “우리 강아지가 좀 이상해요.”처럼 부족한 정보를 입력합니다.
9. `create_daily_care_note`가 임시 위험도, 확인된 정보, 부족한 정보 질문, 오늘 관리 행동을 안내합니다.
10. 호스트 Agent가 가족방 캡처에서 읽어낸 텍스트를 `chatText`로 전달합니다.
11. `summarize_pet_chat_for_vet`가 식욕, 변 상태, 구토 여부, 활동량, 음식 언급, 시간 흐름을 정리하고 병원 상담용 요약을 생성합니다.
12. 호스트 Agent가 변 사진에서 관찰한 색과 형태를 텍스트로 전달합니다.
13. `record_pet_photo_observation`이 관찰 텍스트를 위험도와 병원 상담용 `vetShareCard`로 구조화합니다.
14. 보호자가 “부산 진구 근처 동물병원 알려줘.”라고 입력합니다.
15. `find_nearby_animal_hospitals`가 지역 기반 동물병원 후보와 방문 전 전화 확인 안내를 제공합니다.
16. 보호자가 병원 검색을 요청하지 않은 상황에서는 `dogProfile`의 기존 병원 연락과 `vetCallScript`를 먼저 안내하고 병원 검색을 자동 제안하지 않습니다.
17. 보호자가 “몽이가 밥을 반만 먹고 변이 묽어. 집밥 원료는 어떤 기준으로 봐야 해?”라고 입력합니다.
18. `create_daily_care_note`가 기존 식단 유지, 저지방 원료 비교, 한 가지씩 소량 확인, 다음 배변 기록 기준과 주의 원료를 제공합니다.
